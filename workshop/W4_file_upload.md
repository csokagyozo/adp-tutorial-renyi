# Working with files

Most ChatGPT-style interfaces allow you to upload files, although the exact file types and size limits depend on the system. In mathematical work, useful files may include papers, notes, LaTeX source files, images, tables, spreadsheets, slides, or draft manuscripts. After uploading a file, you can ask questions about its content instead of copying long passages into the chat.

Typical tasks include: summarizing the structure of a paper, extracting definitions or notation, checking a proof attempt, comparing two versions of a draft, improving exposition, finding where a concept is introduced, or turning rough notes into a clearer explanation. For LaTeX files, ChatGPT can often help with rewriting, formatting, debugging errors, or making notation more consistent.

Uploaded files should be used deliberately. A large file may consume substantial context or credit, and the model may not always focus on the part that is most relevant to your question. It is usually better to start with a structural question, such as “Please summarize the main sections and identify where Lemma 3 is used,” and then continue with more focused local questions about a particular definition, proof, table, or passage.

As with ordinary chat answers, file-based answers must be checked. ChatGPT may miss details, misunderstand notation, or summarize a technical argument too confidently. For serious mathematical use, treat file upload as a way to navigate and discuss documents more efficiently, not as a substitute for reading and verification.

Before uploading files, also check the privacy and confidentiality section, especially for unpublished work, referee reports, student data, or personal data.

## Tools beyond text

ChatGPT is primarily a text-based assistant, but some models and interfaces can also use tools. Depending on the system, these may include file analysis, code execution, data analysis, web search, image generation, or other specialized functions. The exact tool set is not universal: it depends on the interface, the selected model, permissions, and sometimes also on the current usage limits.

For mathematical work, tools can be useful when ordinary text is not enough. Code execution can help test examples, search small cases, check computations, or make plots. File analysis can help navigate papers, notes, tables, or drafts. Image generation may be useful for sketches, diagrams, or presentation material, though mathematically precise figures usually need additional checking or manual correction.

It is important to treat tool outputs as assistance, not as final results. Code may contain bugs, plots may be misleading, web search results may be incomplete, and generated images may contain mathematical or typographical errors. Tools can speed up exploration, but the researcher remains responsible for verification.
